# Audit i18n Deep — TopCryptoCards — Juillet 2026

> Audit réalisé avec inspection du code source + navigation live sur topcryptocards.eu en 5 langues.

## 🔴 CRITIQUE — Visible utilisateur, casse l'expérience non-FR

### BUG #1 — `feature_*` keys manquants dans `extrasLabels.ts`
**Impact** : Toutes langues, toutes cartes  
**Symptôme observé** : Sidebar "Extra Benefits" affiche `"Feature cashback boost"`, `"Feature free account"`, `"Feature instant card"` — texte brut anglais  
**Fichier** : `src/i18n/extrasLabels.ts`  
**Cause** : 7 clés DB présentes dans les migrations ne sont pas dans le dictionnaire de labels :

| Clé DB | Manquant dans extrasLabels.ts | Traductions dans cards.json |
|--------|-------------------------------|----------------------------|
| `feature_free_account` | ✗ | ✅ FR/DE/ES/IT/EN disponibles |
| `feature_cashback_boost` | ✗ | ✅ FR/DE/ES/IT/EN disponibles |
| `feature_instant_card` | ✗ | ✅ FR/DE/ES/IT/EN disponibles |
| `feature_staking_rewards` | ✗ | ✗ pas dans cards.json |
| `feature_lounge_access` | ✗ | ✗ pas dans cards.json |
| `feature_insurance` | ✗ | ✗ pas dans cards.json |
| `feature_concierge` | ✗ | ✗ pas dans cards.json |

**Fix** : Ajouter ces 7 clés à `src/i18n/extrasLabels.ts` avec traductions 5 langues.

### BUG #2 — ReviewPage : contenu 100% en français sur toutes les langues non-FR
**Impact** : Toutes cartes × DE/ES/IT/EN  
**Symptôme observé** : `/en/reviews/crypto-com-card` affiche badge, keyStats, pros, cons, sections, verdict… tout en français  
**Cause** : Le script `scripts/generate-review-translations.mjs` n'a pas terminé avec succès. `src/data/cardReviewsI18n.ts` est vide (`REVIEW_I18N = {}`).  
**Fix** : Exécuter `node scripts/generate-review-translations.mjs` (le script a été corrigé avec retry + skip-already-done). Valider les 96 traductions.  
**Note** : Le code ReviewPage lui-même est correct — il utilise bien `getReviewI18n(slug, lang)` avec fallback.

---

### BUG #3 — `recommend.ts` : toutes les raisons/défauts en français
**Impact** : Page Recommendation résultats × DE/ES/IT/EN  
**Fichier** : `src/utils/recommend.ts`  
**Symptôme** : Un utilisateur DE voit "Aucun frais annuel", "Retraits ATM gratuits", "Staking de X € requis pour le cashback maximum", etc.  
**Lignes concernées** :
```typescript
reasons.push(`Cashback élevé jusqu'à ${c.cashbackPremium}%`)  // ligne 60
reasons.push(`${effectiveCashback}% de cashback sans staking requis`)  // ligne 66
reasons.push('Aucun frais annuel')  // ligne 72
reasons.push('Aucun staking requis')  // ligne 83
reasons.push('Retraits ATM gratuits')  // ligne 92
reasons.push('Simple à prendre en main, sans staking ni frais')  // ligne 99
reasons.push('Avantages voyage inclus')  // ligne 106
drawbacks.push(`Staking de ${c.stakingRequired} € requis...`)  // ligne 123
drawbacks.push(`Frais annuels de ${c.annualFees} €`)  // ligne 126
drawbacks.push('Non disponible en France')  // ligne 129
drawbacks.push('Retraits ATM payants')  // ligne 132
```
**Fix** : Créer un map `REASONS_MAP[lang]` avec traductions pour toutes les strings dynamiques et statiques.

### BUG #4 — `market_restrictions` en français sur toutes les pages non-FR
**Impact** : CardDetail × DE/ES/IT/EN, cartes avec restrictions (ex: OKX, certaines cartes Binance)  
**Symptôme observé** : Badge rouge affiche `"FR — Non disponible — restrictions AMF"` en anglais et allemand  
**Cause** : La valeur de la clé `fr` dans `market_restrictions` est stockée en français en DB. `CardDetail.tsx` affiche toutes les restrictions de tous les marchés pour toutes les langues.  
**Fichier** : `src/pages/CardDetail.tsx` lignes 405-418  
**Fix** : Soit localiser le texte de restriction (créer un map de valeurs → traductions), soit masquer les restrictions non pertinentes pour le marché courant.

## 🟠 ÉLEVÉ — Visible utilisateur, dégradation significative

### BUG #5 — BrandPage : badge de review seulement en FR
**Impact** : BrandPage × DE/ES/IT/EN  
**Fichier** : `src/pages/BrandPage.tsx` ligne 1021  
**Code** : `{lang === 'fr' && review.badge && (...)}`  
**Symptôme** : Badge "La plus connue", "Meilleur cashback" etc. visible uniquement sur `/fr/marques/...`  
**Fix** : Utiliser `getReviewI18n(slug, lang)?.badge ?? (lang === 'fr' ? review.badge : undefined)` — ou afficher le badge FR pour toutes les langues comme fallback acceptable jusqu'à ce que les traductions soient disponibles.

---

### BUG #6 — brandConfig.ts : noms de villes/pays en français
**Impact** : BrandPage "Siège social" × DE/ES/IT/EN  
**Fichier** : `src/data/brandConfig.ts`  
**Valeurs françaises** : Singapour, Royaume-Uni, Suisse, Dubaï, États-Unis, Autriche, Allemagne, Pologne, Non communiqué  
**Toutes les 20 marques sont affectées**  
**Fix** : Créer une map de localisation pour les noms de pays/villes, ou rendre `hq` un objet `Record<lang, string>` par marque.

---

### BUG #7 — ComparisonPage : intro/verdict FR par défaut pour 12/15 paires
**Impact** : ComparisonPage × DE/ES/IT  
**Fichier** : `src/data/comparisonContent.ts` + `src/pages/ComparisonPage.tsx` ligne 237  
**Cause** : `fr_intro ?? specificContent.fr_intro` — si `de_intro` n'existe pas, fallback vers le texte FR  
**Statistiques** :
- FR + EN : 15 paires (toutes)  
- DE : 3 paires seulement  
- ES : 1 paire seulement  
- IT : 1 paire seulement  
**Fix** : Changer le fallback de `fr_intro` vers `undefined` (utiliser le texte générique traduit), OU compléter les 15 paires en DE/ES/IT.

---

### BUG #8 — BlogPost : breadcrumb "Home" hardcodé en anglais
**Impact** : BlogPost × FR/DE/ES/IT (page FR voudrait "Accueil")  
**Fichier** : `src/pages/BlogPost.tsx` ligne 185  
**Code** : `{ label: 'Home', href: \`/${lang}\` }`  
**Fix** : Ajouter map `HOME_LABEL = { fr: 'Accueil', de: 'Startseite', es: 'Inicio', it: 'Home', en: 'Home' }` et utiliser `HOME_LABEL[lang]`.

---

### BUG #9 — ThematicPage : `€/an` hardcodé en français
**Impact** : ThematicPage, grille de cartes × DE/ES/IT/EN  
**Fichier** : `src/pages/ThematicPage.tsx` ligne 892  
**Code** : `` `${card.annual_fees} €/an` ``  
**Symptôme** : Une carte affiche "50 €/an" sur la page anglaise au lieu de "€50/year"  
**Fix** : Ajouter à la map `L` existante : `annual_suffix: { fr: '€/an', de: '€/Jahr', es: '€/año', it: '€/anno', en: '€/year' }` et utiliser `t('annual_suffix')`.

---

### BUG #10 — BrandPage : liste pros vide sur pages non-FR
**Impact** : BrandPage × DE/ES/IT/EN quand `seo.pros` est undefined  
**Fichier** : `src/pages/BrandPage.tsx` ligne 1067  
**Code** : `{(seo.pros ?? (lang === 'fr' ? review.pros : [])).slice(0, 2)...}`  
**Symptôme** : Section "Points forts" vide sur toutes les pages non-FR sans override SEO  
**Fix** : Utiliser `review.pros` pour toutes les langues comme fallback acceptable (le contenu des pros est identique pour toutes les langues actuellement), soit utiliser `getReviewI18n(slug, lang)?.pros`.

**Bonne nouvelle** : La structure i18n de base est solide. Les 330 clés JSON sont complètes. ThematicPage, ComparisonPage et CryptoPage ont du contenu localisé correct. Les bugs restants sont des points de données manquants ou des fallbacks FR mal configurés, pas des problèmes architecturaux.
