# Comparison Pages — les pages qui gagnent les requêtes « best tool »
À lancer après la baseline citations.md. Inputs : résultats citations.md, your-site/brand-voice.md.

---

Les pages comparaison/alternatives gagnent une part disproportionnée des citations IA. Nous avons déjà ~30 paires A-vs-B éditoriales + 120 indexables — l'angle ici est différent :

Depuis nos résultats de citations, lister chaque buyer query où la page d'un concurrent gagne. Pour chacune :

1. La page à construire : « [carte] alternatives » (manquant chez nous ×10 marques ×5 langues — chantier connu), « best [category] for [profil] » (ex. best crypto card for travel in Europe), ou une A-vs-B qui n'est pas dans l'allowlist mais que l'IA cite.
2. La table de comparaison nécessaire : critères de VRAIS acheteurs (frais réels, cashback après conditions, staking, dispo par pays, IBAN, licence MiCA) — depuis notre base Supabase, pas des bullets marketing.
3. FAQ avec les buyer questions; comparaison de prix; schema.
4. Ce qui rendrait notre page 2x meilleure que celle qui gagne — le nommer précisément (notre atout par défaut : données datées + dispo par pays + 5 langues), ou dire que l'écart est ingagnable et changer de cible.

RÈGLE D'HONNÊTETÉ : si une carte concurrente est meilleure pour un profil, l'écrire. Honnête = citable. Les puff pieces sont ignorées par les modèles ET par les acheteurs.

OUTPUT : table — requête à gagner | page à construire | format | effort (S/M/L) | priorité. Puis l'outline de la page n°1, prête à écrire.
